# WorkSpace v14 — soft delete всего, серия одной строкой на Ганте, UX-фикс

> Дата: 2026-07-22. Прод: миграции только additive (0010, 0011 — только
> ADD COLUMN NULL, ломающего нет).

## 1. Поле новой подзадачи

После Enter поле создания закрывается (создал одну — поле исчезло);
серийный ввод убран. Постоянная строка «Новая задача…» внизу дерева
остаётся — она не «появляется».

## 2. Soft delete всего (миграция 0010)

`deleted_at TEXT NULL` в tasks, projects, task_types, people, roles.

- Все DELETE-ручки помечают `deleted_at` вместо удаления строк; API и
  фронт не меняются (та же семантика «исчезло из списков»).
- Задачи: каскад по поддереву — пометка всех живых потомков.
- Проект: правило «удалять только пустой» считает живые задачи и живые
  под-проекты.
- Справочники: отвязки остаются как были (тип/исполнитель/роль →
  SET NULL у ссылающихся), сама запись помечается. Связки
  project_members чистятся жёстко (это ссылки, не данные).
- Все SELECT фильтруют `deleted_at IS NULL` (включая CTE поддеревьев,
  сиблингов, MAX(position), проверки существования): мёртвые записи
  невидимы и не участвуют в нумерации/валидации.
- UI восстановления нет — страховка на уровне данных (вернуть можно
  руками в sqlite). Отдельная «Корзина» — POSTPONED, триггер: первый
  случай реальной потери.

## 3. Серия на Ганте одной строкой (миграция 0011)

`tasks.series_id INTEGER NULL` — якорь серии:

- при установке правила повтора задаче без series_id он становится
  равным её id;
- спавн (done и разовый перенос) наследует series_id;
- снятие правила series_id не трогает — история остаётся связанной.

Гант: задачи одной серии внутри проекта схлопываются в одну строку.
Носитель — живое вхождение (или последнее по дате, если живого нет);
прошлые done-вхождения — приглушённые ромбики на той же дорожке;
призраки будущего — как раньше. Строки и поддеревья прочих вхождений
серии на Ганте не показываются. Неделя и дерево не меняются.

Масштабируемость: series_id индексируем запросами по мере надобности;
на нём же позже строится «история серии» и групповые операции.

## 4. Тесты

Store: soft delete задач (каскад, невидимость, счётчик), проекта
(пустота по живым), типа/человека/роли (списки, отвязки); series_id
(установка правила, наследование при спавне и one-переносе).
Фронт: Enter закрывает поле подзадачи; Гант-группировка — юнит на
функцию группировки серий.
