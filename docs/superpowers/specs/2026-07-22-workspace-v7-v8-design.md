# WorkSpace v7+v8 — календарь и полировка; Тип и Команда

> Дата: 2026-07-22. Решения пользователя: календарь «свой + 3 чипа»;
> Тип — один, опционально; исполнитель — один, опционально.

## v7 — фиксы и UI

1. **Баг «задачи не удаляются»** — root cause: ConfirmButton сбрасывал
   взвод по mouseleave (реальная мышь между кликами почти всегда чуть
   уходит с кнопки → второй клик заново взводил), плюс тихий 3с-таймер.
   Фикс: сброс только по таймеру (4с) и Escape; взведённое состояние —
   явно красное.
2. **Свой DatePicker** вместо DateMenu и нативного `<input type=date>`:
   чипы «Сегодня · Завтра · Пн» (след. понедельник), мини-календарь в
   стиле space (mono-caps «ИЮЛЬ 2026 ◂ ▸», сетка пн–вс, сегодня —
   контуром, выбранный — заливкой, диапазон план→«по» — asoft-фоном),
   внизу «по…» (переключает выбор конца, только у плана) и «Снять».
   Размещение: в инспекторе и модале — **инлайн-раскрытие** под чипами
   (ничего не обрезается), в строках дерева — поповер.
3. **Resizable-колонки** в «Проектах»: ручки на границах
   сайдбар|дерево|инспектор, pointer-drag, клампы (сайдбар 180–400,
   инспектор 240–440), ширины в localStorage.
4. **Корзина** (инлайн-SVG, штриховая) вместо ✕ на удалениях: проекты в
   сайдбаре/архиве, кнопка в инспекторе.
5. **Описание**: без рамки, автовысота (height=scrollHeight), без
   внутреннего скролла и resize-уголка.

## v8 — Тип и Команда

Модель (миграция 0006):

- `task_types(id, name, position, created_at, updated_at)` — глобальный
  справочник; `tasks.type_id NULL REFERENCES task_types` (SET NULL при
  удалении типа — вручную в транзакции).
- `people(id, name, color, position, created_at, updated_at)`;
  `tasks.assignee_id NULL REFERENCES people` (при удалении человека его
  задачи теряют исполнителя). Пустой исполнитель = «делаю я».

API: `/api/types` и `/api/people` (GET/POST/PATCH/DELETE); задачи
принимают `typeId`/`assigneeId` (Opt, null значим).

UI:

- Инспектор/модал: секции «Тип» (меню существующих + создание нового
  прямо в меню + «убрать») и «Исполнитель» (люди кружками-инициалами +
  «я»).
- Строка дерева: mono-caps бейдж типа (`DEV`) и кружок-инициалы после
  названия; карточка недели — те же метки в строке крошки.
- Раздел **«Команда»** (`/team`, хоткей `4`): список людей — цвет
  (палитра проектов), имя (rename дабл-кликом), корзина с подтверждением
  (задачи остаются, исполнитель снимается), добавление внизу.

Тесты: store (CRUD типов/людей, SET NULL при удалении, назначение на
задачу, валидации), api-контракты; smoke по всем пунктам.
