# WorkSpace v9 — типы-эмодзи, роли, участники проекта, компактная карточка

> Дата: 2026-07-22. Одобрено пользователем, включая решения: раздел
> «Типы» отдельно (хоткей 5); одна роль у человека; стек участников — в
> шапке дерева проекта; подсказки жестов — видимые в UI.

## 1. Модель (миграция 0007)

- `task_types.emoji TEXT NOT NULL DEFAULT ''`.
- `roles(id, name, position, created_at, updated_at)`;
  `people.role_id INTEGER NULL REFERENCES roles(id)` (удаление роли —
  SET NULL вручную в транзакции).
- `project_members(project_id, person_id, PRIMARY KEY(project_id,
  person_id))` — участники проекта (несколько). Удаление человека чистит
  его членства; удаление проекта — членства проекта.

## 2. API

- `/api/types`: + `emoji` в POST/PATCH/GET.
- `/api/roles`: GET/POST/PATCH/DELETE.
- `/api/people`: + `roleId` (PATCH — Opt, null значим; GET отдаёт).
- `GET /api/members` → все пары; `PUT /api/projects/{id}/members`
  `{personIds: []}` — полная замена состава (транзакция).

## 3. UI

### Раздел «Типы» (`/types`, хоткей `5`)

Список: кнопка-эмодзи (поповер: пресеты 💻🧪🤝📞📝🐛🎨🚀🔧📊 + свободное
поле для любого смайла) · имя (rename дабл-кликом) · корзина
(подтверждение вторым кликом; задачи теряют тип). Добавление внизу.
Видимая строка-подсказка про ⌃⌘Space. Создание типов из карточки задачи
удаляется — только выбор.

### Бейджи типов

Везде вместо текста — эмодзи (имя в тултипе): строка дерева, карточка и
«продолжение» недели, Гант — справа от названия задачи в колонке имён.
Тип без эмодзи — первые 2 буквы имени mono-caps.

### Карточка задачи (инспектор и модал)

- Убираются заголовки секций и инлайн-раскрытия; под названием — одна
  линия чипов: план · дедлайн · тип · исполнитель; описание сразу под ней.
- Все меню — маленькие попапы с **fixed-позиционированием по якорю**
  (общий AnchoredPopover; не обрезаются в модале, кламп к viewport):
  план → календарь с переключалкой «День | Диапазон» (диапазон: первый
  клик — начало, второй — конец); дедлайн → просто календарь; тип и
  исполнитель → миниатюрные списки.
- Крошки: только в модале недели (`showCrumb`); в инспекторе и попапе
  Ганта строка проекта/пути не показывается.

### Команда и участники

- Блок «Роли» на странице команды: создание, rename дабл-кликом,
  корзина; у человека — чип роли с мини-списком, подпись роли рядом с
  именем.
- Шапка дерева проекта: стек аватарок участников + кнопка ＋ → попап-
  чеклист людей (полная замена состава).

### Подсказки

- Кнопка «?» в шапке — поповер-шпаргалка: хоткеи 1–5 и T, дабл-клик —
  переименование, второй клик — подтверждение удаления, drag-жесты
  дерева/Ганта.
- Контекстные строки: «Типы» — ⌃⌘Space; «Команда» — дабл-клик.

## 4. Тесты

Store: emoji у типов, роли CRUD + SET NULL, членства (replace,
чистка при удалении человека/проекта). API-контракты. Vitest: без новых
модулей (календарь-диапазон — логика в компоненте, проверяется smoke).
Smoke: раздел «Типы», смайлы во всех трёх видах, компактная карточка с
fixed-попапами в модале Ганта (ничего не обрезается), роли, участники,
«?»-шпаргалка.
