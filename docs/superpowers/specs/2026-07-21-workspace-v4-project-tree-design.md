# WorkSpace v4 — откат каскада, дерево проектов, архив, доводка Ганта

> Дата: 2026-07-21. Дельта к v3, по запросу пользователя.

## 1. Откат каскада done (отменяет §4 спеки v2)

Пользователь передумал: **никаких автоматических синхронизаций done** —
всё руками. Удаляются все четыре правила v2 (авто-закрытие предков,
авто-открытие при снятии/создании/переносе). Счётчик «сделано/всего»
остаётся. Ручная отметка любого узла — как и была.

## 2. Дерево проектов

- `projects.parent_id INTEGER NULL REFERENCES projects(id)` — произвольная
  вложенность; циклы запрещает сервер (422). `position` — в рамках
  родителя.
- Задачи может иметь проект любого уровня.
- Сайдбар — дерево с шевронами (раскрытие в localStorage); drag строк:
  середина — под-проектом, верх/низ — сиблингом (как у задач).
- Счётчик у проекта — несделанные задачи **всего поддерева проектов**.
- Гант: раскрытие проекта показывает сначала под-проекты (рекурсивно),
  затем задачи; отступы по глубине.

## 3. Удаление и архив проектов

- `projects.archived INTEGER NOT NULL DEFAULT 0`.
- **Удалить можно только пустой** проект (нет задач и нет под-проектов),
  иначе 422. В сайдбаре у строки одна из двух иконок: ✕ (пустой) или
  «в архив» (непустой).
- **Архивация рекурсивна вниз** (весь поддерев проектов), разархивация —
  тоже; исключает состояние «неархивный ребёнок внутри архивного».
- Архивные: скрыты из основного списка сайдбара (секция «Архив» внизу,
  свёрнута по умолчанию; там кнопки «вернуть» и ✕ для пустых), их задачи
  скрыты из недели (колонки и плашка просрочки) и из Ганта; на Ганте —
  тумблер «архив» в шапке.
- Запреты: перенос проекта/задачи в архивный проект — 422; создание
  задачи в архивном — 422; quick-add не предлагает архивные.

## 4. Перенос задач между проектами (UI)

Drag узла дерева задач на строку проекта в сайдбаре — задача с поддеревом
уезжает в корень того проекта (сервер перекрашивает `project_id`
поддерева; механика v2). Внутри дерева перенос работает как раньше.

## 5. Запрет «дедлайн раньше плана»

Если у задачи заданы обе даты — сервер требует `due_on >= scheduled_on`
(422 «дедлайн раньше запланированного дня»); проверка на create и на
patch по финальному состоянию. Осознанный минус зафиксирован: сценарий
«план после сорванного дедлайна» станет невозможен; при необходимости
ослабим до предупреждения.

## 6. Неделя

- Тумблер «Сб–Вс» в шапке: скрыть/показать выходные (5 или 7 колонок),
  состояние в localStorage.
- Задачи архивных проектов не показываются (колонки, плашка, счётчики).

## 7. Гант — доводка

- Числа у **каждого** дня в шапке (сегодня — акцентом), не только у
  понедельников.
- Тонкая пунктирная вертикальная сетка-разделитель дней (SVG-паттерн,
  читаемый в обеих темах) поверх подсветки выходных.
- Подпись месяца — sticky внутри своего сегмента (не обрезается при
  горизонтальном скролле).
- Тумблер «архив»: показывает архивные проекты (приглушённо).

## 8. API-дельта

- `projectJSON` + `parentId`, `archived`; `POST /api/projects` принимает
  `parentId`; `PATCH` — `parentId` (null=в корень), `archived` (bool,
  рекурсивно; ответ — все затронутые), позиция в scope родителя.
- `DELETE /api/projects/{id}`: 422 при задачах или под-проектах.
- `PATCH /api/tasks/{id}` + `projectId`: перенос в корень указанного
  проекта (поддерево перекрашивается, позиция — в конец корней).

## 9. Тесты

Store: снятие каскада (done больше не трогает предков; создание не
открывает), дерево проектов (циклы, sibling-позиции в scope родителя,
reparent), архив рекурсивный, запреты удаления/переносов/создания,
валидация due>=scheduled, перенос задачи через projectId. Фронт: селектор
видимых дней недели, агрегированный счётчик проекта. Smoke: всё из §2–7.
