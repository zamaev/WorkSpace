> перенос с выбором «только эту | всю серию», призраки будущих
> вхождений, копирование подзадач со сброшенными галочками, на старте
> только еженедельная сетка по дням недели.

## 1. Фикс: drop на полоску-неделю не назначал дату

Регресс v12: capture-слушатель `drop` прятал полоску до того, как
событие доходило до `onDrop` ячейки (микротаски между слушателями →
React размонтировал ячейку → обработчик размонтированного не зовётся).
Фикс: в capture-обработчике drop внутри `.dragweek` игнорируется —
полоску прячет сам onDrop ячейки после назначения даты.

## 2. Повторяющиеся задачи

### Модель (миграция 0009, additive)

- `tasks.repeat TEXT NULL` — JSON `{"kind":"weekly","days":[1,4]}`,
  дни ISO (1=пн … 7=вс), непустые, уникальные, отсортированы.
- Инварианты: повтор требует `scheduled_on`; несовместим с `end_on`
  (встреча — один день); 422 при нарушении.
- В структуре всегда ОДНА живая задача серии; правило живёт у неё.

### Механика (сервер, транзакции)

- **Done** (переход done 0→1 у задачи с repeat): текущая помечается
  done и правило с неё снимается; создаётся копия — title, description,
  project, parent, type, assignee, repeat; position в конец скоупа;
  `scheduled_on` = ближайшее вхождение строго после
  max(старая дата, сегодня); дедлайны и day_position не копируются.
  Подзадачи копируются рекурсивно (title/description/type/assignee) со
  сброшенным done и без дат. Повторный PATCH done=true — no-op (нет
  перехода). Снятие done не трогает спавненную копию.
- **Перенос** (PATCH scheduledOn у задачи с repeat, поле
  `repeatScope`):
  - `"one"` — разовый: спавнится следующая по расписанию (от старой
    даты, тот же алгоритм и копирование, что у done, но текущая НЕ
    done); правило переезжает в новую, у текущей снимается, дата
    меняется на новую;
  - `"series"` (и по умолчанию) — просто смена даты, правило остаётся:
    расписание едет от новой даты.

### API

- GET: `repeat: {kind, days} | null` (объект, не строка — toJSON
  вручную). PATCH: `repeat` (Opt, null снимает), `repeatScope`.
- POST create без repeat (ставится из карточки).

### UI

- Чип «повтор» в карточке задачи: попап с 7 тогглами дней недели +
  «снять»; подпись чипа «↻ пн · чт».
- Значок ↻ в строке дерева и на карточке недели.
- Смена даты повторяющейся (неделя, полоска, Гант, календарь, «на
  сегодня») — DataProvider сам перехватывает patch со scheduledOn и
  показывает мини-диалог «только эту | всю серию | отмена», дополняя
  патч repeatScope. Места вызова не меняются.
- Призраки будущих вхождений: полупрозрачные, не таскаются, клик
  открывает задачу. Неделя — в видимых днях; Гант — контурные ромбики
  до конца шкалы, горизонт 90 дней от плановой даты.

## 3. Тест-слой (регрессия чиненого)

- @testing-library/react + jsdom + стаб DataTransfer; мок fetch с
  in-memory данными.
- Компонентные: drop на ячейку полоски назначает дату (день и диапазон
  с сохранением длины); полоска исчезает после drop на ячейку И после
  переноса в подзадачу; drop в дерево — reparent; клик по карточке
  недели открывает модал, чекбокс — нет; повторный клик по фильтру
  снимает его; диалог «только эту | серию» появляется только у
  повторяющихся.
- Юниты: nextOccurrence/occurrencesBetween (границы недели, cap),
  формат подписи дней.
- Store-тесты: валидация правила, done-спавн (перенос полей, подзадачи
  undone, идемпотентность, un-done), перенос one/series.
