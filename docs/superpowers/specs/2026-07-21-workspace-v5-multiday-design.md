# WorkSpace v5 — многодневные задачи, двухнедельный вид, фиксы

> Дата: 2026-07-21. Дельта к v4. Подтверждено пользователем: многодневная
> задача видна в каждом дне недели своего диапазона.

## 1. Модель: `tasks.end_on`

- Новое поле `end_on TEXT NULL` — «работаю по…» (включительно). Диапазон
  работы: `scheduled_on … end_on`.
- Правила: `end_on` без `scheduled_on` — 422; `scheduled_on <= end_on`;
  снятие `scheduled_on` обнуляет и `end_on` (диапазон без начала не
  существует). Дедлайн независим от `end_on` (работа может быть
  запланирована за дедлайн — это честно показывает проблему); правило
  `due_on >= scheduled_on` из v4 сохраняется.
- Миграция 0005 — один ALTER.

## 2. Семантика трёх дат

| Поле | Смысл | В неделе | На Ганте |
|---|---|---|---|
| `scheduledOn` | день начала работы | полновесная карточка | начало полосы / ромб |
| `endOn` | последний день работы | «продолжения» в каждом дне диапазона | конец сплошной полосы |
| `dueOn` | крайний срок | чип «до …» | контурный флажок-ромб + пунктирный хвост от конца работы |

## 3. Неделя

- Сетка: число колонок = числу видимых дней (баг «пустое место без
  выходных» уходит).
- **Многодневная задача**: в день `scheduledOn` — обычная карточка
  (+бейдж «1/N»); в остальные дни диапазона — облегчённое «продолжение»:
  чекбокс + приглушённое название + бейдж «k/N», рендерится группой выше
  обычных задач дня, в счётчик дня не входит, в day_position не участвует.
- Drag любой карточки многодневки двигает **весь диапазон** (началу
  присваивается целевой день, длина сохраняется, endOn сдвигается той же
  дельтой; клиент шлёт оба поля).
- **Тумблер «2 недели»**: под текущей строкой — вторая с неделей+7
  (заголовки дней в ячейках уже есть); drag между строками работает
  нативно; листание двигает обе. Состояние в localStorage.
- Назначение диапазона: в DateMenu задачи добавляется вторая строка
  «по…» (календарь) — рядом с «Снять»; также каждое «продолжение» не
  предлагает действий диапазона (только чекбокс/детали).

## 4. Гант — задачи

- Сплошная полоса — только диапазон работы `scheduledOn→endOn`; drag
  середины двигает диапазон, края — начало/конец работы.
- Только `scheduledOn` — ромб (как раньше).
- `dueOn` — **контурный** ромб-флажок (бордер цветом проекта, прозрачная
  заливка), таскается отдельно; если дедлайн позже конца работы — тонкая
  пунктирная линия-хвост от конца работы до флажка.
- Проекты не меняются (их полоса start→due — «жизнь проекта»).

## 5. Фиксы

- **Палитра проектов**: поповер налезал на строки. Root cause подтвердить:
  подозрение — important-модификаторы записаны v3-синтаксисом (`!left-0`),
  Tailwind v4 требует суффикс (`left-0!`), т.е. переопределения молча не
  применялись. Полный аудит всех `!`-классов проекта (grep, точечное
  чтение, замена), для поповера — обычный класс `.popover-left` вместо
  important-переопределений.
- Сетка недели без выходных растягивается (см. §3).

## 6. Тесты

Store: end_on валидации (без начала, порядок, авто-обнуление при снятии
плана), совместимость с due-правилом. Фронт: селектор «продолжений»
(spanTasksOn), сдвиг диапазона при drag, timeline-математика хвоста.
Smoke: многодневка в неделе (обе строки при «2 неделях»), drag полосы и
флажка на Ганте, палитра во вложенном сайдбаре, обе темы.
