# WorkSpace v6 — инспектор задачи, чистка взаимодействий

> Дата: 2026-07-22. Чисто фронтовая итерация (API не меняется). Решения
> согласованы: инспектор — постоянная третья колонка; полоски цвета в
> дереве остаются.

## 1. Инспектор задачи (общий компонент TaskDetails)

Содержимое: чекбокс + название (редактируемое поле) · крошка
проект/путь · три даты чипами с меню (план + «работаю по», дедлайн) ·
описание (textarea, автосохранение по blur) · кнопка «Удалить»
(подтверждение вторым кликом) · в модальном варианте — ссылка «в дереве →».

- **Проекты**: постоянная третья колонка (~300px). Пусто → подсказка
  «выбери задачу». Клик по строке дерева = выбор (подсветка строки);
  повторный клик не снимает; ✕ в панели закрывает выбор; удаление задачи
  сбрасывает.
- **Неделя/Гант**: клик по названию карточки/строки открывает тот же
  TaskDetails модалом (идиома sheet из space: оверлей, центр, Escape и
  клик по оверлею закрывают). Inline-детали из карточек недели и
  Link-переход из имён на Ганте удаляются.

## 2. Чистка строк

- Дерево: клик по названию = выбор (не rename!); rename — двойной клик
  (шорткат) или поле в инспекторе. Hover-действия строки: только ＋
  (подзадача). ◷/⚑/≡/✕ из строки удаляются; чипы существующих дат
  остаются кликабельными с меню; всё остальное — в инспекторе.
- Сайдбар: одиночный клик = выбор проекта, двойной по активному = rename.
  ✕/архив остаются; window.confirm заменяется «подтверждением вторым
  кликом» (кнопка краснеет в «точно?», второй клик выполняет, сброс через
  3с или по уходу мыши). Тот же паттерн — в инспекторе задачи.
- Инпуты добавления: строка-инпут задач получает полоску цвета проекта и
  ту же геометрию, что настоящая строка (без сдвига после Enter); инпут
  проекта — колонку шеврона и полоску-заглушку.

## 3. Мелкие правки UX

- DateMenu: вместо «вся текущая неделя» — «Сегодня · Завтра», остаток
  текущей недели (от послезавтра), затем вся следующая неделя.
- Quick-add недели: placeholder «＋ в {проект}…»; точка-кнопка остаётся.
- Гант: имена задач выравниваются с именем проекта (единая базовая линия
  отступов: базовый сдвиг = шеврон+полоска, глубина — шагом); строки
  задач без единой даты — приглушены.
- Хоткей-подсказки: title на кнопках навигации («Проекты — 1» и т.п.).

## 4. Тесты

tsc/vitest прежние + smoke: выбор строки → инспектор, редактирование
названия/описания/дат из панели, модал в неделе и на Ганте, добавление
задач без сдвига, подтверждение вторым кликом, выравнивание Ганта.
