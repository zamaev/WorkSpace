# WorkSpace v3 — дедлайны, даты проектов, Гант

> Дата: 2026-07-21. Дельта к v2. Решения согласованы с пользователем
> (все четыре развилки — по рекомендациям).

## 1. Новые поля

- `tasks.due_on TEXT NULL` (`YYYY-MM-DD`) — **дедлайн**: до какой даты
  сделать. Независим от `scheduled_on` (план «в какой день делаю»);
  жёсткой валидации между ними нет — это решение пользователя.
- `projects.start_on TEXT NULL`, `projects.due_on TEXT NULL` — старт и
  дедлайн проекта. **Задаются руками** (не вычисляются из задач); если обе
  заданы — сервер требует `start_on <= due_on` (422).
- Миграция 0003 — три `ALTER TABLE ADD COLUMN`, без пересборки таблиц.
- Инвариант v2 «задача всегда в проекте» уже выполняется (`project_id NOT
  NULL`) — в v3 без изменений.

## 2. API

- `taskJSON` + `dueOn`; `POST /api/tasks` принимает `dueOn`; `PATCH` —
  `dueOn` с семантикой null = снять (Opt).
- `projectJSON` + `startOn`, `dueOn`; `PATCH /api/projects/{id}` принимает
  оба с null-семантикой.

## 3. Дерево и неделя

- Строка узла: после чипа плана — чип дедлайна «⚑ Пт 25»; просроченный
  (`dueOn < сегодня && !done`) — цветом `--over`. Клик — тот же DateMenu
  (заголовок «Дедлайн»); hover-действие «⚑» когда дедлайна нет.
- Карточка недели: под названием мелкий чип «до Пт 25» (красный при
  просрочке) рядом с крошкой.
- Плашка «Просрочено» — **две секции**: «Сорван дедлайн» (dueOn < сегодня;
  действия: «дедлайн: сегодня», «снять дедлайн») и «Не сделано в
  запланированный день» (scheduledOn < сегодня; действия прежние). Задача,
  попадающая в обе, показывается только в секции дедлайнов.

## 4. Гант (`/gantt`, хоткей `3`, пункт меню «Гант»)

Лэйаут: слева колонка названий (фикс. ширина), справа — горизонтально
скроллируемая шкала дней (день = константа px), сверху метки месяцев и
чисел, вертикальная линия «сегодня». Диапазон: min/max всех дат ± запас,
минимум — окно вокруг сегодня. Реализация своя (grid + pointer events),
без Гант-библиотек.

Строки-проекты (порядок сайдбара):

- Обе даты → полоса цвета проекта (мягкая заливка + кант). Drag середины —
  сдвиг обеих дат; drag краёв — start/due по отдельности (курсор resize).
- Одна дата → ромб-маркер; drag двигает её.
- Без дат → строка без фигуры, в шкале кнопка «＋ даты» (ставит
  start=сегодня, due=+13 дней — дальше тянуть).
- Шеврон раскрывает задачи проекта (иерархия с отступами; состояние —
  localStorage).

Строки-задачи:

- Обе даты (`scheduledOn`+`dueOn`) → тонкая полоса цвета проекта;
  drag середины — сдвиг обеих, края — по отдельности.
- Одна дата → ромб (drag двигает её). Без дат → имя слева, шкала пустая;
  первичное назначение дат задачи — из дерева/недели (на Ганте не делаем,
  чтобы не изобретать жест «создать полосу»).
- Done-задачи приглушены (opacity), как вторичное в языке space.

Все правки дат — оптимистичные PATCH существующими ручками.

## 5. Тесты

- Store: set/unset `due_on` задач; даты проекта, 422 при start>due;
  миграция 0003 поверх v2-данных.
- Фронт: unit на модуль шкалы Ганта (день↔пиксель, диапазон, месячные
  сегменты) и на новые селекторы просрочки (дедлайн против плана,
  дедупликация «в обеих категориях»).
- Smoke: назначить дедлайны, увидеть обе секции плашки, Гант — драг полосы
  проекта (обе даты сдвинулись), resize края, ромб задачи, раскрытие
  проекта, «＋ даты», обе темы.
